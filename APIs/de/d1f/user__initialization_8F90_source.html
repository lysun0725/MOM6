247 !> \class user_initialization
248 !!
249 !! By Robert Hallberg, April 1994 - June 2002 *
250 !! *
251 !! This subroutine initializes the fields for the simulations. *
252 !! The one argument passed to initialize, Time, is set to the *
253 !! current time of the simulation. The fields which are initialized *
254 !! here are: *
255 !! u - Zonal velocity in m s-1. *
256 !! v - Meridional velocity in m s-1. *
257 !! h - Layer thickness in m. (Must be positive.) *
258 !! G%bathyT - Basin depth in m. (Must be positive.) *
259 !! G%CoriolisBu - The Coriolis parameter, in s-1. *
260 !! GV%g_prime - The reduced gravity at each interface, in m s-2. *
261 !! GV%Rlay - Layer potential density (coordinate variable), kg m-3. *
262 !! If ENABLE_THERMODYNAMICS is defined: *
263 !! T - Temperature in C. *
264 !! S - Salinity in psu. *
265 !! If BULKMIXEDLAYER is defined: *
266 !! Rml - Mixed layer and buffer layer potential densities in *
267 !! units of kg m-3. *
268 !! If SPONGE is defined: *
269 !! A series of subroutine calls are made to set up the damping *
270 !! rates and reference profiles for all variables that are damped *
271 !! in the sponge. *
272 !! Any user provided tracer code is also first linked through this *
273 !! subroutine. *
274 !! *
275 !! Forcing-related fields (taux, tauy, buoy, ustar, etc.) are set *
276 !! in MOM_surface_forcing.F90. *
277 !! *
278 !! These variables are all set in the set of subroutines (in this *
279 !! file) USER_initialize_bottom_depth, USER_initialize_thickness, *
280 !! USER_initialize_velocity, USER_initialize_temperature_salinity, *
281 !! USER_initialize_mixed_layer_density, USER_initialize_sponges, *
282 !! USER_set_coord, and USER_set_ref_profile. *
283 !! *
284 !! The names of these subroutines should be self-explanatory. They *
285 !! start with "USER_" to indicate that they will likely have to be *
286 !! modified for each simulation to set the initial conditions and *
287 !! boundary conditions. Most of these take two arguments: an integer *
288 !! argument specifying whether the fields are to be calculated *
289 !! internally or read from a NetCDF file; and a string giving the *
290 !! path to that file. If the field is initialized internally, the *
291 !! path is ignored. *
292 !! *
293 !! Macros written all in capital letters are defined in MOM_memory.h. *
294 !! *
295 !! A small fragment of the grid is shown below: *
296 !! *
297 !! j+1 x ^ x ^ x At x: q, CoriolisBu *
298 !! j+1 > o > o > At ^: v, tauy *
299 !! j x ^ x ^ x At >: u, taux *
300 !! j > o > o > At o: h, bathyT, buoy, tr, T, S, Rml, ustar *
301 !! j-1 x ^ x ^ x *
302 !! i-1 i i+1 At x & ^: *
303 !! i i+1 At > & o: *
304 !! *
305 !! The boundaries always run through q grid points (x). *
306 end module user_initialization
